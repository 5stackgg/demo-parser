# demo-parser

Tiny HTTP wrapper around [markus-wa/demoinfocs-golang](https://github.com/markus-wa/demoinfocs-golang) that the 5stack `api` calls to extract metadata from a `.dem` file:

- `total_ticks`, `tick_rate`, `map_name` (header)
- `round_ticks` — `[{round, start_tick, end_tick}]` filtered to the real
  match (warmup + knife excluded)

The web client uses these to drive a tick-based scrubber + round-jump buttons against a CS2 instance running the demo (`+playdemo …`) inside `game-streamer`.

## Endpoint

```
POST /parse
{
  "match_map_demo_id": "<uuid>",
  "demo_url": "<pre-signed s3 GET>"
}

200 OK
{
  "total_ticks": 120000,
  "tick_rate": 64.0,
  "map_name": "de_mirage",
  "round_ticks": [
    {"round": 1, "start_tick": 1234, "end_tick": 9876},
    ...
  ]
}
```

The parser does **not** persist anything itself — the api writes the result back to `match_map_demos`. The pre-signed URL is generated by the api with a generous (~30min) expiry so the parser can retry transient S3 hiccups without a round trip.

## Future: 2D playback

The same `internal/parser` package is the natural home for a `ParseFrames` pass that emits per-tick `{players[], grenades[], bomb}` records for browser-side 2D replay. Spec lives in the plan file at `~/.claude/plans/now-that-we-have-foamy-newt.md` (§ "Future 2D Playback").

## Build / run

```sh
go build ./cmd/server
LISTEN_ADDR=:8080 ./server
```

In Kubernetes this runs as a Deployment + ClusterIP Service named `demo-parser` in the `5stack` namespace; the api reaches it via `DEMO_PARSER_URL` (default `http://demo-parser.5stack.svc.cluster.local:8080`).
